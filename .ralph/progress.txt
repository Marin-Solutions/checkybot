## Codebase Patterns
<!-- Patterns accumulate here -->
- Package-facing Checkybot routes should use the shared `api.key` Bearer-token middleware and set the request user resolver, which lets Form Request ownership checks keep working without a separate package auth code path.
- Package-managed monitoring records should be archived, not hard-deleted, when sync prunes them so history remains queryable in Filament and tests can assert archived visibility.
- Health event routing should reuse `NotificationSetting` scopes plus `WebsiteServicesEnum` inspection values instead of introducing a parallel notification configuration path.
- Application rollup status should be computed from non-archived components only, with `danger > warning > healthy` precedence so archived incidents do not affect current state.
- Application detail pages should expose package-managed checks through dedicated `Project` relation managers that remove the soft-delete global scope, so sync-pruned records remain visible as archived history.
